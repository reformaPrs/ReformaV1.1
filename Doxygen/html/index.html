Компиляция
Действия для компиляции нового приложения
  1. Изменить в файле config.h:
    • Дату окончания работы программы (EXPIRATION_DATE)
    • Режим работы программы для специалиста или родителя (PROF_VERSION)
    • Последний серийный номер прибора со старой прошивкой (LAST_OLD_SERIAL_NUMBER)
  2. Скомпилировать с помощью Qt 6.2.3 MinGw 64-bit на релиз
  3. Скопировать из папки релиза Reforma.exe в папку Reforma_Release
  4. скомпилировать скрипт соответствующий режиму работы программы для генерации установщика
Приложение написано на основе MioReference.doc
Главные изменения
  1. VH пакеты переделали в AH (сделал чел ещё до меня, тупо заменил букву для контроля версий)

  2. При отправке данных о парах передаются пакеты в следующем порядке:

    • AH_PACKET_DATA_0
    • AH_PACKET_DATA_1
    • AH_PACKET_DATA_2
    • AH_PACKET_DATA_3
    • AH_PACKET_DATA_4

    При этом данные не запишутся без последнего. В первых 4х пакетах информация о состоянии пар, а в последнем информация о получасе и интервале. То есть в AH_PACKET_DATA_4 после байта с номером пакета идёт байт с длиной итервала в секундах (1-20), потом байт с получасом (0 - первый, 1 - второй), потом байт с номером пакета получаса в порядке убывания, заканчивая 1.

  3. Переход на версию Qt 6.2.3 (старая была 5.13), но при небольших корректировках можно скомпилировать под Qt 5.15.2
